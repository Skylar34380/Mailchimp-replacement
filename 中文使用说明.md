# 邮件订阅管理系统中文使用说明

这个项目的目标是：**搭建一个通用的自托管邮件订阅管理系统**。

它不绑定任何具体网站。你可以把它接到普通官网、落地页、Webflow、WordPress、Next.js 应用，或任何能提交 HTTP 表单的网站。

```text
网站或应用订阅表单
  -> subscribe-api
  -> Listmonk
  -> SMTP 服务商
  -> 订阅者邮箱
```


## 整体流程

```text
1. 启动 Listmonk
2. 创建邮件列表，例如 Newsletter Updates
3. 拿到这个列表的 UUID
4. 配置 subscribe-api
5. 网站表单提交到 subscribe-api
6. subscribe-api 把订阅者写入 Listmonk
7. 在 Listmonk 后台创建 newsletter 并发送
8. 在 Listmonk 后台查看打开率、点击率、退订情况
```

## 第一步：准备环境

电脑需要安装：

- Docker Desktop
- Node.js 20 或更高版本

如果只是用 Docker Compose 跑完整系统，主要需要 Docker Desktop。

## 第二步：进入项目目录

```bash
cd newsletter-stack
```

复制环境变量文件：

```bash
cp .env.example .env
```

## 第三步：启动系统

先打开 Docker Desktop，等它显示 Docker 正在运行。

然后运行：

```bash
docker compose up -d --build
```

启动后访问：

```text
Listmonk 后台: http://localhost:9000
Subscribe API 健康检查: http://localhost:8787/health
```

## 第四步：创建邮件列表

进入：

```text
http://localhost:9000
```

在 Listmonk 后台创建一个 public list，例如：

```text
Newsletter Updates
```

创建后复制这个 list 的 UUID。

UUID 不是付费授权码。它只是这个邮件列表的唯一编号。

例如：

```text
Newsletter Updates
UUID: ce13e971-c2ed-4069-bd0c-240e9a9f56f9
```

## 第五步：配置 subscribe-api

打开：

```text
newsletter-stack/.env
```

把：

```text
LISTMONK_LIST_UUIDS=replace-with-public-list-uuid
LISTMONK_DRY_RUN=true
```

改成：

```text
LISTMONK_LIST_UUIDS=你刚刚复制的-list-uuid
LISTMONK_DRY_RUN=false
```

然后重启：

```bash
docker compose up -d --build
```

## 第六步：测试订阅 API

可以用这个命令模拟网站表单提交：

```bash
curl -X POST http://localhost:8787/api/subscribe \
  -H 'Content-Type: application/json' \
  --data '{
    "email": "reader@example.com",
    "firstName": "Reader",
    "lastName": "One",
    "source": "website-home",
    "consentText": "I agree to receive newsletter updates."
  }'
```

如果成功，会看到：

```json
{
  "ok": true,
  "message": "Subscription received.",
  "dryRun": false
}
```

然后去 Listmonk 后台的 Subscribers 页面，应该能看到这个邮箱。

## 第七步：接入真实网站

把你的网站订阅表单提交地址改成你的 subscribe-api 地址。

本地测试时是：

```text
http://localhost:8787/api/subscribe
```

真正上线后应该是类似：

```text
https://subscribe.your-domain.com/api/subscribe
```

推荐表单字段：

```text
email
firstName
lastName
pageUrl
source
consentText
```

可选分群字段：

```text
company
role
interests
```

subscribe-api 会把这些字段清洗后发给 Listmonk。

## 第八步：配置 SMTP 发信

Listmonk 本身负责管理邮件列表和 campaign，但真正发信需要 SMTP 服务。

可选：

- Amazon SES：便宜，适合长期使用
- Postmark：简单稳定，价格更高
- Mailgun：常见选择
- SMTP2GO：配置相对简单

在 Listmonk 后台的 Settings 里配置 SMTP。

配置好之后，就可以在 Listmonk 里创建 campaign 并发送。

## 第九步：查看打开率和点击率

Listmonk 可以在 campaign 页面查看：

- 发送数量
- 打开 / views
- 点击 / clicks
- 退订 / unsubscribes
- bounce
- subscriber 状态

注意：打开率不是 100% 准确。Apple Mail Privacy Protection 和图片屏蔽会影响所有邮件工具。

点击率通常比打开率更可靠。

## 如果邮件进垃圾箱怎么办？

请看：

```text
邮件送达率与垃圾邮件防护指南.md
```

重点是：

- 不要自己搭 SMTP
- 用 Amazon SES / Postmark / Mailgun / SMTP2GO
- 配置 SPF、DKIM、DMARC
- 使用清晰的 unsubscribe
- 不导入已退订或 bounced 的联系人
- 新系统小批量 warm up
- 监控 bounce、unsubscribe、spam complaint

## 生产部署建议

推荐结构：

```text
newsletter.your-domain.com   -> Listmonk
subscribe.your-domain.com    -> subscribe-api
```

部署方式：

- 一台小 VPS + Docker Compose
- Caddy / Nginx 做 HTTPS
- Postgres 数据库定期备份
- SMTP 用 Amazon SES 或 Postmark
