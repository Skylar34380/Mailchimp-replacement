# 邮件送达率与垃圾邮件防护指南

这个文档解决一个核心问题：

```text
如果不用 Mailchimp，自己用 Listmonk 群发邮件，会不会进垃圾箱？
```

答案是：**有可能**。不管用 Mailchimp、Listmonk、Amazon SES、Postmark、Mailgun，群发邮件都有进垃圾箱风险。

但只要按下面步骤做，风险可以大幅降低。

## 1. 不要自己搭 SMTP 发信服务器

Listmonk 负责：

- 邮件列表
- campaign
- 模板
- 退订
- 打开率 / 点击率
- subscriber 状态

真正发邮件应该交给专业 SMTP 服务：

- Amazon SES：便宜，适合长期
- Postmark：简单稳定，价格更高
- Mailgun：常见选择
- SMTP2GO：配置友好

不要用自己 VPS 直接发邮件。新 IP 没有声誉，很容易进垃圾箱。

推荐：

```text
Listmonk + Amazon SES
```

或者更省心：

```text
Listmonk + Postmark
```

## 2. 必须配置 SPF、DKIM、DMARC

这三个是发信域名认证。

### SPF

告诉邮箱服务商：

```text
哪些邮件服务器可以代表我的域名发邮件
```

### DKIM

给邮件加数字签名。

收件方可以验证：

```text
这封邮件确实由我的域名授权发出，内容没有被篡改
```

### DMARC

告诉收件方：

```text
如果 SPF/DKIM 验证失败，应该怎么处理
```

Google 官方 sender guidelines 要求发信者设置 SPF/DKIM，bulk sender 还需要 DMARC。Amazon SES 文档也明确建议 SPF、DKIM、DMARC 一起使用来保护和认证发信域名。

参考：

- Google sender guidelines: https://support.google.com/a/answer/81126
- Amazon SES DMARC: https://docs.aws.amazon.com/ses/latest/dg/send-email-authentication-dmarc.html
- Postmark deliverability guides: https://postmarkapp.com/guides/deliverability

## 3. 用子域名发营销邮件

不要直接用主域名发营销邮件。

推荐：

```text
网站主域名: ommelb.com.au
营销邮件域名: mail.ommelb.com.au
退信/return-path: bounce.ommelb.com.au
Listmonk 后台: newsletter.ommelb.com.au
订阅 API: subscribe.ommelb.com.au
```

这样即使营销邮件声誉出问题，也不会直接伤害主域名。

## 4. 一定要有清晰退订

每封营销邮件必须有：

- 明显的 unsubscribe 链接
- List-Unsubscribe header
- 发件人身份和联系地址

澳洲商业邮件也要求：

- 有 consent
- 标明 sender identity
- 提供可用退订方式
- 退订请求要及时处理

不要隐藏退订链接。隐藏退订只会让用户点“Report spam”，这比退订更伤发信声誉。

## 5. 不要导入已经退订的人

从 Mailchimp 迁移时：

```text
subscribed   -> 可以导入 active/confirmed
unsubscribed -> 不要重新激活
cleaned      -> 不要发送
bounced      -> 不要发送
```

如果把退订/无效邮箱重新导入，垃圾投诉和 bounce 会升高，很快影响送达率。

## 6. 清理名单

导入前建议做：

- 去重
- 删除明显错误邮箱
- 保留 consent 来源
- 保留订阅时间
- 保留 Mailchimp 标签/分组
- 不导入 role-based 邮箱，除非确实有 consent

常见 role-based 邮箱：

```text
info@
admin@
sales@
support@
contact@
```

这些邮箱不一定不能发，但投诉和无效率通常更高，要谨慎。

## 7. 新系统要慢慢 warm up

不要第一天就群发给所有人。

推荐节奏：

```text
第 1 天: 发给 20-50 个最可信联系人
第 2-3 天: 发给 100-200 个
第 4-7 天: 发给 300-500 个
之后再逐步扩大
```

如果名单很小，比如几百人，也要先小批量测试。

## 8. 控制邮件内容

房产 newsletter 建议：

- 标题真实，不夸张
- 不要全大写标题
- 不要太多感叹号
- 不要只放一张大图
- 图片和文字要平衡
- 每个项目有简短说明
- 链接使用自己的域名或可信域名
- 避免短链接
- 避免附件

不要写：

```text
AMAZING DEAL!!!
CLICK NOW!!!
LIMITED TIME ONLY!!!
```

可以写：

```text
New off-market townhouse opportunities in Melbourne
```

## 9. 监控指标

每次 campaign 后看：

- bounce rate
- unsubscribe rate
- complaint rate
- open rate
- click rate

注意：open rate 不完全准确。Apple Mail Privacy Protection 和图片屏蔽会影响打开率，Mailchimp 也一样。

更可靠的是：

```text
click rate
reply / enquiry
unsubscribe rate
bounce rate
spam complaint rate
```

## 10. 推荐告警阈值

如果出现这些情况，要暂停发送：

```text
bounce rate > 2%
unsubscribe rate > 1%
spam complaint rate > 0.1%
open rate 突然大幅下跌
大量 Gmail/Yahoo/Outlook 退信
```

暂停后检查：

- DNS 是否正确
- SPF/DKIM/DMARC 是否通过
- 是否导入了错误名单
- 是否发送太快
- 内容是否像垃圾邮件
- 链接域名是否可疑

## 11. DNS 配置 Checklist

上线前必须完成：

```text
[ ] 选择 SMTP provider
[ ] 验证发信域名
[ ] 配置 SPF
[ ] 配置 DKIM
[ ] 配置 DMARC
[ ] 配置 Return-Path / bounce domain
[ ] 配置 Listmonk SMTP
[ ] 测试发给 Gmail
[ ] 测试发给 Outlook
[ ] 测试发给 Yahoo
[ ] 检查邮件 header 是否 SPF/DKIM/DMARC pass
[ ] 确认 unsubscribe 链接有效
[ ] 确认 Listmonk 能记录 opens/clicks
```

## 12. 建议 DMARC 起步策略

刚开始不要直接设置最严格。

可以从：

```text
v=DMARC1; p=none; rua=mailto:dmarc@your-domain.com
```

观察一段时间后，再逐步升级：

```text
p=quarantine
```

最后再考虑：

```text
p=reject
```

如果不熟悉 DNS，先用 SMTP provider 给的配置，不要自己乱写。

## 13. 生产环境推荐

对 OMM 来说，推荐：

```text
Wix 网站继续保留
Listmonk 管理邮件列表
Postmark 或 Amazon SES 负责 SMTP
subscribe-api 接 Wix 表单
```

最省心：

```text
Listmonk + Postmark
```

最低成本：

```text
Listmonk + Amazon SES
```

## 14. 出现进垃圾箱怎么办

按这个顺序排查：

1. 检查 SPF/DKIM/DMARC 是否 pass。
2. 检查 From 域名和 DKIM/Return-Path 是否 alignment。
3. 检查是否用了新域名或新发信 IP，需要 warm up。
4. 检查是否导入了无 consent 或旧名单。
5. 检查 bounce/unsubscribe/spam complaint。
6. 减少发送量，先发给活跃用户。
7. 改邮件标题和内容，减少促销感。
8. 确认退订链接明显且可用。
9. 如果用 SES，检查 SES reputation dashboard。
10. 如果用 Postmark/Mailgun，查看 provider 的 deliverability 报告。

## 15. 我们这个项目已经为防垃圾做了什么

`subscribe-api` 已经包括：

- email 格式校验
- honeypot 反垃圾字段
- IP rate limit
- consent text 记录
- source/page 记录
- user-agent/origin/referer 记录
- 不直接暴露 Listmonk 后台 API

下一步可以继续加：

- Cloudflare Turnstile
- email verification
- double opt-in
- domain DNS checker
- bounce/complaint webhook 处理
- 自动暂停高风险 campaign
